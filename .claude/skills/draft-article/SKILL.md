AIに不慣れなビジネスパーソン向けの記事を、元記事URLをもとに生成し、Notion に下書きとして自動保存するスキル。

## 入力形式

```
url: <元記事のURL>
file: <テキストファイルのパス>  ← URLが取得できない場合はこちら
category: 導入ヒント | 実践ガイド（必須）
points: |
  - 強調したいポイント1
  - 強調したいポイント2
（任意）
```

- `url` または `file` のいずれかが必須。両方指定された場合は `file` を優先する。
- `category` が未指定の場合は確認する。

## 実行手順

### Step 1: 入力を確認する

引数から `url` または `file`、`category`、`points` を読み取る。
`category` が未指定の場合は「導入ヒント」か「実践ガイド」のどちらかを確認する。
`url` と `file` の両方が指定された場合は `file` を優先する。

### Step 2: 元記事を取得・読み込む

引数に `url:` が指定されている場合、WebFetch でコンテンツを取得する。

取得に失敗した場合（認証エラー・有料記事・302リダイレクト等）は、以下のエラーメッセージを表示してそこで停止する：

```
❌ 元記事を取得できませんでした。
記事の本文をテキストでここに貼り付けてください。
貼り付け後、もう一度 /draft-article を実行してください。
```

引数に `file:` が指定されている場合（例: `file:C:\Users\tkoba\Downloads\article.txt`）、そのファイルを Read ツールで読み込む。

**sourceUrl の扱い:**
- `url:` が指定されている場合は、その値を `sourceUrl` に使う（`file:` と併用していても同様）
- `file:` のみで `url:` がない場合は、`sourceUrl` は null にする

### Step 3: 記事本文と概要文を生成する

以下の条件で日本語記事を生成する。

**スタイル:**
- ですます調
- 対象読者: AIをまだ使い始めていないビジネスパーソン
- 文字数: 5000字前後（元記事の分量に応じて4000〜6000字の範囲で調整してよい）
- 元記事の内容から大きく逸れない（補足・修正・読みやすく再構成する）
- 専門用語は初出時にカッコ内で簡単な説明を付ける（例: LLM（大規模言語モデル））

**タイトルの付け方:**
元記事の内容に合わせて、以下のパターンから最も適切なものを選ぶ：
- 課題起点型：「〜で困っていませんか？〜する方法」
- 具体的な成果型：「〜が〇〇から〇〇に。〜の実践的な使い方」
- 疑問解消型：「〜って結局何に使えるの？〜が今すぐ試せる〜」
- 数字入り型：「〜で変わった5つのこと。〜の実例から学ぶ」

タイトルにAI専門用語を使う場合は、わかりやすい言い換えも添える。

**構成:**
```
# タイトル

## はじめに
（導入。以下の内容を丁寧に・ボリュームをもって書く）
- この記事で取り上げるテーマを平易な言葉で説明する
- なぜ今このテーマが重要なのか、ビジネスの文脈で背景を伝える
- 読者（AIを使い始めていないビジネスパーソン）が「自分ごと」として感じられるよう、
  具体的な業務シーンや悩みに引き寄せた書き方にする
- この記事を読むことで何が得られるか（学べること・できるようになること）を明示する
- 500字前後を目安に書く

## [見出し1]
...

[IMAGE: ○○のイメージ図]  ← 画像があると理解しやすい箇所に挿入

## [見出し2]
...

## まとめ
- ポイント1
- ポイント2
- ポイント3
```

**画像指示:**
- 本文の理解を助ける箇所に `[IMAGE: 説明]` を挿入する（2〜4箇所）
- 説明は「○○のフロー図」「○○と○○の比較図」など具体的に

**points が指定された場合:**
- 指定されたポイントを記事内で意識して盛り込む

**概要文（excerpt）の生成:**
- 記事全体の内容を50〜80字で要約する
- ですます調で書く
- 一覧カードに表示されるため、読者の興味を引く表現にする
- 例：「AIエージェントを使えば、仕事の受注から納品まで自動化できます。ビジネスに活かすための基本的な考え方を解説します。」

### Step 4: 生成した記事を JSON に変換する

以下の形式で `draft-article.tmp.json`（プロジェクトルート直下）に保存する。

```json
{
  "title": "記事タイトル",
  "category": "導入ヒント",
  "excerpt": "50〜80字の概要文。一覧カードに表示される。",
  "sourceUrl": "https://... または null",
  "blocks": [
    { "type": "heading_2", "text": "はじめに" },
    { "type": "paragraph", "text": "導入文1段落目（テーマ説明・背景）..." },
    { "type": "paragraph", "text": "導入文2段落目（自分ごと化・この記事で得られること）..." },
    { "type": "heading_2", "text": "見出し1" },
    { "type": "paragraph", "text": "本文..." },
    { "type": "paragraph", "text": "本文続き..." },
    { "type": "image_placeholder", "text": "[IMAGE: ○○のフロー図]" },
    { "type": "heading_2", "text": "見出し2" },
    { "type": "paragraph", "text": "本文..." },
    { "type": "bulleted_list_item", "text": "箇条書き1" },
    { "type": "bulleted_list_item", "text": "箇条書き2" },
    { "type": "bulleted_list_item", "text": "箇条書き3" },
    { "type": "image_placeholder", "text": "[IMAGE: ○○の比較図]" },
    { "type": "heading_2", "text": "見出し3" },
    { "type": "paragraph", "text": "本文..." },
    { "type": "heading_2", "text": "まとめ" },
    { "type": "bulleted_list_item", "text": "ポイント1" },
    { "type": "bulleted_list_item", "text": "ポイント2" },
    { "type": "bulleted_list_item", "text": "ポイント3" }
  ]
}
```

※ このサンプルは構造の例示。実際は5000字前後に必要なブロック数を過不足なく生成すること。各見出しセクションは複数の段落で十分に掘り下げ、具体例・背景・補足説明を加えてボリュームを出す。見出し・段落・箇条書きは内容に応じて増減してよい。

ブロック種別:
- `paragraph`: 通常の文章
- `heading_2`: 大見出し（##）
- `heading_3`: 小見出し（###）
- `bulleted_list_item`: 箇条書き
- `numbered_list_item`: 番号付きリスト
- `image_placeholder`: 画像挿入指示（`[IMAGE: 説明]`）
- `quote`: 引用
- `divider`: 区切り線

### Step 5: Notion に下書きページを作成する

```bash
node scripts/create-notion-draft.js draft-article.tmp.json
```

を実行する。実行後、`draft-article.tmp.json` を削除する：

```bash
rm draft-article.tmp.json
```

### Step 6: 完了を報告する

- Notion の下書きページ URL を表示する
- ユーザーが Notion で行うべき作業を案内する：
  1. 本文の確認・修正
  2. スラッグの入力（例: `ai-chatgpt-introduction`）
  3. 概要文の確認・修正（自動生成済み）
  4. アイキャッチ画像の作成・添付
  5. `[IMAGE: 説明]` を参考に画像を手動で作成・挿入
  6. ステータスを「公開」に変更してサイトに反映
