# ユースケースページ設計書

**プロジェクト名:** AIレベルチェッククイズアプリ — /usecases 機能
**作成日:** 2026-03-30
**バージョン:** 1.0

---

## 1. 概要

AIのビジネス活用に関する読み物コンテンツを提供するページ。
クイズで自分のレベルを知った後、具体的な活用方法を学べる導線として機能する。
将来的には講演・研修オファーの獲得にもつなげる。

### ページ構成

| URL | 役割 |
|-----|------|
| `/usecases` | 記事一覧（カテゴリフィルタ付き） |
| `/usecases/[slug]` | 記事詳細 |

### カテゴリ

| カテゴリ | 内容 |
|---------|------|
| 導入ヒント | AIを現在の仕事にどう導入できるかの記事 |
| 実践ガイド | 実際の活用事例・具体的な手順の記事 |

## 3. コンテンツ管理フロー

### 3.1 記事作成フロー（半自動）

```
① /draft-article スキルを呼び出す
   入力：元記事URL ＋ 伝えたいポイント（任意）
       ↓
② Claude が記事の下書きを生成
   - 記事テンプレートに沿った構成
   - 画像挿入箇所に [IMAGE: 説明] を自動挿入
       ↓
③ Notion に下書きページを自動作成
       ↓
④ ユーザーが Notion でレビュー・編集
   - 本文の修正
   - [IMAGE: 説明] を参考に手動で画像を作成・挿入
       ↓
⑤ ステータスを「公開」に変更
       ↓ Notion オートメーション → Webhook
⑥ 同期 API がサイトに反映
```

### 3.2 再編集フロー

```
ステータスを「再編集中」に変更
    ↓ サイトは古い版を表示継続
Notion 上で本文を編集
    ↓
ステータスを「公開」に変更
    ↓ Webhook → 最新版に更新
```

### 3.3 ステータス定義

| ステータス | サイト表示 | 意味 |
|-----------|----------|------|
| 下書き | 非表示 | 作成中 |
| 公開 | 表示（最新） | 公開済み |
| 再編集中 | 表示（旧版） | 公開済みだが修正中 |
| 公開停止 | 非表示 | 一時的に非表示 |

## 4. Notion データベース構成

### プロパティ一覧

| プロパティ | 種類 | 備考 |
|-----------|------|------|
| 名前 | タイトル | 記事タイトル |
| カテゴリ | セレクト | 導入ヒント / 実践ガイド |
| 概要文 | テキスト | 一覧カードに表示する説明文 |
| アイキャッチ | ファイル | サムネイル画像 |
| スラッグ | テキスト | URL用（例: ai-introduction-tips） |
| 元記事URL | URL | 参考にした記事のリンク |
| ステータス | セレクト | 下書き / 公開 / 再編集中 / 公開停止 |

### ページ本文

Notion のページ本文（ドキュメント形式）に記事を記述する。
- 見出し・本文・箇条書きは通常通り Notion で記述
- 画像は Notion の画像ブロックで本文内に挿入
- アイキャッチ画像はプロパティに添付

## 5. Supabase テーブル設計

テーブル名：`usecases`

```sql
CREATE TABLE usecases (
  id            serial PRIMARY KEY,
  notion_id     text UNIQUE NOT NULL,
  title         text NOT NULL,
  slug          text UNIQUE NOT NULL,        -- URL用スラッグ
  category      text NOT NULL,               -- '導入ヒント' | '実践ガイド'
  excerpt       text,                        -- 一覧カード用の概要文
  thumbnail_url text,                        -- Supabase Storage の永続URL
  content       jsonb NOT NULL DEFAULT '[]', -- Notion ブロックを変換した JSON
  source_url    text,                        -- 元記事URL
  is_active     boolean DEFAULT true,
  published_at  timestamp,
  created_at    timestamp DEFAULT now(),
  updated_at    timestamp DEFAULT now()
);
```

Storage バケット：`usecases-images`（public）
- Notion からダウンロードした画像を永続保存する

## 6. 同期 API 設計（`/api/sync-article`）

Notion ステータス変更 → Webhook → 同期 API の処理：

```
1. Notion API でページのプロパティを取得
2. Notion API でページのブロック一覧を取得
3. ブロック内の画像を処理：
   a. Notion の一時URLから画像をダウンロード
   b. Supabase Storage にアップロード
   c. コンテンツ内のURLを Supabase Storage の永続URLに差し替え
4. プロパティのアイキャッチ画像も同様に Supabase Storage へ移行
5. Supabase の usecases テーブルへ upsert
```

**ステータスと is_active のマッピング:**

| Notion ステータス | is_active |
|-----------------|----------|
| 公開 | true |
| 再編集中 | true（旧コンテンツを維持） |
| 下書き / 公開停止 | false |

## 7. /draft-article スキル設計

### 7.1 概要

元記事URLをもとに、AIに不慣れなビジネスパーソン向けの読みやすい記事を生成し、Notionに下書きとして自動保存するスキル。

### 7.2 実装ファイル

| ファイル | 役割 |
|---------|------|
| `.claude/commands/draft-article.md` | Claude Code スキル定義 |
| `scripts/create-notion-draft.js` | Notion API でページ作成するスクリプト |

### 7.3 呼び出し方

```
/draft-article url:https://... category:導入ヒント points:|
  - 中小企業向けの事例を強調
  - 難しい用語を避ける
```

- `url`（必須）: 元記事のURL
- `category`（必須）: `導入ヒント` または `実践ガイド`
- `points`（任意）: 強調・補足したいポイントを箇条書き
- URLのみでも起動可能（category は対話で確認）

### 7.4 処理フロー

```
① /draft-article 起動
     ↓
② Claude が元記事URLのコンテンツを取得・読み込み
     ↓
③ 記事本文を生成（ターミナルに表示）
   - ですます調、AI未経験ビジネスパーソン向け
   - 800〜1200文字（元記事の分量に応じて調整）
   - 元記事の内容から大きくそれない
   - 画像挿入箇所に [IMAGE: 説明] を挿入
     ↓
④ scripts/create-notion-draft.js を自動実行
     ↓
⑤ Notion に下書きページ作成 → ページURLを表示
```

### 7.5 記事スタイルガイド

| 項目 | 方針 |
|------|------|
| 文体 | ですます調 |
| 対象読者 | AIを使い始めていないビジネスパーソン |
| 文字数 | 800〜1200字（元記事に応じて柔軟に） |
| 元記事との関係 | 補足・修正・読みやすく再構成（大きく逸れない） |
| 画像指示 | `[IMAGE: ○○のイメージ図]` を本文中の適切な箇所に挿入 |
| 専門用語 | 初出時にカッコ内で簡単な説明を付ける |

### 7.6 Notion ページ作成仕様

- **スクリプト:** `scripts/create-notion-draft.js`
- **必要な環境変数:** `NOTION_API_KEY`、`NOTION_DATABASE_ID`（`.env.local`）
- **作成されるページのプロパティ:**

| プロパティ | 設定値 |
|-----------|--------|
| 名前（タイトル） | 生成した記事タイトル |
| カテゴリ | 入力した category |
| 元記事URL | 入力した url |
| ステータス | 下書き |
| スラッグ | 空（Notion上で手動入力） |
| 概要文 | 空（Notion上で手動入力） |
| アイキャッチ | 空（Notion上で手動追加） |

- **ページ本文:** 生成した記事本文をNotionブロック（見出し・段落・箇条書き）として書き込む

### 7.7 実行後のユーザー作業

Notion上で以下を手動で行う：
1. 本文の確認・修正
2. スラッグの入力（URL用、例: `ai-chatgpt-introduction`）
3. 概要文の入力（一覧カード表示用、50〜80字）
4. アイキャッチ画像の作成・添付
5. `[IMAGE: 説明]` の箇所に画像を手動で挿入
6. ステータスを「公開」に変更 → サイトに反映

- 公開頻度：週2本
- 作成フロー：/draft-article スキル → Notion 編集 → 公開
