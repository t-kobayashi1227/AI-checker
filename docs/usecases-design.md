# ユースケースページ設計書

**プロジェクト名:** AIレベルチェッククイズアプリ — /usecases 機能
**作成日:** 2026-03-30
**バージョン:** 1.0

---

## 1. 概要

AIのビジネス活用に関する読み物コンテンツを提供するページ。
クイズで自分のレベルを知った後、具体的な活用方法を学べる導線として機能する。
将来的には講演・研修オファーの獲得にもつなげる。

### ページ構成

| URL | 役割 |
|-----|------|
| `/usecases` | 記事一覧（カテゴリフィルタ付き） |
| `/usecases/[slug]` | 記事詳細 |

### カテゴリ

| カテゴリ | 内容 |
|---------|------|
| 導入ヒント | AIを現在の仕事にどう導入できるかの記事 |
| 実践ガイド | 実際の活用事例・具体的な手順の記事 |

## 3. コンテンツ管理フロー

### 3.1 記事作成フロー（半自動）

```
① /draft-article スキルを呼び出す
   入力：元記事URL ＋ 伝えたいポイント（任意）
       ↓
② Claude が記事の下書きを生成
   - 記事テンプレートに沿った構成
   - 画像挿入箇所に [IMAGE: 説明] を自動挿入
       ↓
③ Notion に下書きページを自動作成
       ↓
④ ユーザーが Notion でレビュー・編集
   - 本文の修正
   - [IMAGE: 説明] を参考に手動で画像を作成・挿入
       ↓
⑤ ステータスを「公開」に変更
       ↓ Notion オートメーション → Webhook
⑥ 同期 API がサイトに反映
```

### 3.2 再編集フロー

```
ステータスを「再編集中」に変更
    ↓ サイトは古い版を表示継続
Notion 上で本文を編集
    ↓
ステータスを「公開」に変更
    ↓ Webhook → 最新版に更新
```

### 3.3 ステータス定義

| ステータス | サイト表示 | 意味 |
|-----------|----------|------|
| 下書き | 非表示 | 作成中 |
| 公開 | 表示（最新） | 公開済み |
| 再編集中 | 表示（旧版） | 公開済みだが修正中 |
| 公開停止 | 非表示 | 一時的に非表示 |

## 4. Notion データベース構成

### プロパティ一覧

| プロパティ | 種類 | 備考 |
|-----------|------|------|
| 名前 | タイトル | 記事タイトル |
| カテゴリ | セレクト | 導入ヒント / 実践ガイド |
| 概要文 | テキスト | 一覧カードに表示する説明文 |
| アイキャッチ | ファイル | サムネイル画像 |
| スラッグ | テキスト | URL用（例: ai-introduction-tips） |
| 元記事URL | URL | 参考にした記事のリンク |
| ステータス | セレクト | 下書き / 公開 / 再編集中 / 公開停止 |

### ページ本文

Notion のページ本文（ドキュメント形式）に記事を記述する。
- 見出し・本文・箇条書きは通常通り Notion で記述
- 画像は Notion の画像ブロックで本文内に挿入
- アイキャッチ画像はプロパティに添付

## 5. Supabase テーブル設計

テーブル名：`usecases`

```sql
CREATE TABLE usecases (
  id            serial PRIMARY KEY,
  notion_id     text UNIQUE NOT NULL,
  title         text NOT NULL,
  slug          text UNIQUE NOT NULL,        -- URL用スラッグ
  category      text NOT NULL,               -- '導入ヒント' | '実践ガイド'
  excerpt       text,                        -- 一覧カード用の概要文
  thumbnail_url text,                        -- Supabase Storage の永続URL
  content       jsonb NOT NULL DEFAULT '[]', -- Notion ブロックを変換した JSON
  source_url    text,                        -- 元記事URL
  is_active     boolean DEFAULT true,
  published_at  timestamp,
  created_at    timestamp DEFAULT now(),
  updated_at    timestamp DEFAULT now()
);
```

Storage バケット：`article-images`（public）
- Notion からダウンロードした画像を永続保存する

## 6. 同期 API 設計（`/api/sync-article`）

Notion ステータス変更 → Webhook → 同期 API の処理：

```
1. Notion API でページのプロパティを取得
2. Notion API でページのブロック一覧を取得
3. ブロック内の画像を処理：
   a. Notion の一時URLから画像をダウンロード
   b. Supabase Storage にアップロード
   c. コンテンツ内のURLを Supabase Storage の永続URLに差し替え
4. プロパティのアイキャッチ画像も同様に Supabase Storage へ移行
5. Supabase の usecases テーブルへ upsert
```

**ステータスと is_active のマッピング:**

| Notion ステータス | is_active |
|-----------------|----------|
| 公開 | true |
| 再編集中 | true（旧コンテンツを維持） |
| 下書き / 公開停止 | false |

## 7. /draft-article スキル設計（予定）

**呼び出し方:**
```
/draft-article
```

**入力:**
- 元記事URL（必須）
- 伝えたいポイント（任意・箇条書き）

**出力:**
- 記事テンプレートに沿った下書き本文
- 適切な箇所に `[IMAGE: 〇〇の概念図]` などの画像指示を挿入
- Notion にページとして自動作成（ステータス: 下書き）

**スキルに定義する記事テンプレート:**
```
# タイトル

## 課題・背景
## AI を使った解決策
## 具体的な手順・事例
   [IMAGE: フロー図のイメージ]
## 実践のポイント
## まとめ（3点）
```

## 8. 更新頻度・運用方針

- 公開頻度：週2本

- 画像作成：NotebookLM 等を使って手動作成・Notion に挿入
