# プロダクト要求仕様書（PRD）

**プロジェクト名:** AIレベルチェッククイズアプリ
**作成日:** 2026-03-29
**バージョン:** 1.0

---

## 1. プロダクト概要

### 背景・目的

AIツールの普及により、ビジネスパーソンにも「AIをどう使いこなすか」が求められる時代になっている。
しかし、自分のAI知識がどのレベルにあるかを客観的に把握できる手段は少ない。

本アプリは、AIに関する知識・活用度を10問4択クイズで手軽にチェックし、
レベル判定とアドバイスを提供することで「AIの勉強を始めるきっかけ」を作ることを目的とする。

### ターゲットユーザー

**プライマリ:**
- AIをあまり使用していないビジネスパーソン（20〜50代）
- AIに興味はあるが何から始めればいいかわからない人
- 自分のAI知識を客観的に確認したい人

**セカンダリ:**
- AIを勉強中の学習者
- 社内研修の受講者（将来）

### 解決する課題

| 課題 | 本アプリの解決策 |
|------|----------------|
| 自分のAIレベルがわからない | 10問のクイズでレベルを可視化 |
| AI学習の入口が見つからない | 解説文で学習のきっかけを提供 |
| 難しそうで取り組みづらい | 4択・10問・スマホ対応で手軽に完結 |

## 2. 機能要件

### 2.1 コア機能

| 機能 | 概要 |
|------|------|
| クイズ出題 | AIに関する10問を順番に出題する |
| 4択選択 | 各問題に4つの選択肢を表示する |
| 即時フィードバック | 回答後すぐに正誤と解説を表示する |
| スコア集計 | 10問の正解数を集計する |
| レベル判定 | スコアに応じた4段階のレベルを判定する |
| アドバイス表示 | レベルに応じた学習アドバイスを表示する |

### 2.2 画面要件

#### トップ画面（`/`）
- アプリのタイトルと概要説明
- 「クイズを始める」ボタン
- 所要時間の目安表示（約3分）

#### クイズ画面（`/quiz`）
- 問題番号の表示（例: 1 / 10）
- 進捗バー
- 問題文
- 4つの選択肢ボタン
- 回答後：正誤表示 ＋ 解説文 ＋「次へ」ボタン

#### 結果画面（`/result`）
- スコア表示（例: 7 / 10）
- レベル名と説明
- レベル別アドバイス
- 「もう一度挑戦する」ボタン（`/quiz` へ遷移）
- 「ホームに戻る」ボタン（`/` へ遷移）

### 2.3 レベル判定基準

| スコア | レベル |
|--------|--------|
| 0〜4点 | AI入門者 |
| 5〜7点 | AI活用者 |
| 8〜9点 | AI上級者 |
| 10点   | AIエキスパート |

### 2.4 問題カテゴリ構成（10問）

| カテゴリ | 問題数 | 内容例 |
|---------|--------|--------|
| AIツール紹介 | 3問 | ChatGPT / Claude / Gemini の特徴など |
| 基礎用語 | 3問 | LLM・プロンプト・ハルシネーションなど |
| 使い方・コツ | 2問 | 効果的なプロンプト手法など |
| リスク・倫理 | 2問 | 著作権・情報漏洩リスクなど |

## 5. フェーズ計画

### Phase 1（現在）：静的クイズアプリ
- 問題データは `/data/questions.ts` に直書き
- DB・外部API不使用
- Vercel にデプロイ

### Phase 2（将来）：問題管理の外部化
- **問題作成フロー:**
  1. Claude.ai / ChatGPT で問題文・選択肢・解説を生成
  2. Notion のデータベースに入力
  3. Notion のオートメーション機能でステータス変更を検知 → Webhook を発火
  4. Webhook が同期用 API をたたき Supabase へ書き込み
  5. アプリは Supabase からデータを取得して表示
- **目的:** コード変更なしで問題の追加・編集・差し替えを可能にする

### Phase 3（将来）：拡張機能
- X（旧Twitter）シェア機能（結果・レベルをポストして認知拡大）
- **カテゴリ均等ランダム出題**（問題数を増やし、カテゴリごとに指定数をランダム取得）
- 問題セットの複数化（初級編・上級編など）
- 研修利用向けの管理機能
